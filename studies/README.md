# studies — 한 챕터씩 풀어보는 폴더

abductcli 통합 코드(`src/`)는 따로 있다. 이 폴더는 **문제 한 편씩 풀이**하는 곳.
양적추론 책의 챕터, GLG 본인이 마주친 *스케일이 안 잡히는 장면* — 둘 다 같은 형식으로 푼다.

## 진입 명령 — `/scaleplay`

새 study 한 판은 `/scaleplay <장면 한 줄>` 명령으로 시작한다.

명령 정의: `~/repos/gh/agent-config/commands/scaleplay.md`

이 명령이 박은 어휘 규칙 · 절대 원칙 · 한 판 절차가 이 폴더의 운영 규칙이다.

## 왜 폴더로 나누어 푸나

> 한 인간으로 닿을 수 없는 스케일을 그려보는 일. 혼자는 아니고 에이전트랑 같이.
> 에이전트도 틀릴 수 있으니 인간이 직관을 먼저 박아둬야 한다.

통합 프레임워크는 5개 study 쌓인 뒤에 검토. 그 전엔 복붙으로 시작하고 공통 추출 금지.

## 번호 규칙

- `001~099` — 에릭 재슬로 *양적추론* 책 챕터 풀이 (학습 트랙)
- `101~` — GLG 본인 lived anomaly (banana=101, concert=102, emacs=103, ...)

같은 형식, 다른 트랙.

## 한 study 폴더의 구조

```
studies/NNN-slug/
├── README.md   — anomaly + 첫 직관 + surface + 결과 + 미해결
├── study.clj   — Clojure 풀이 (있으면)
└── data/       — raw + verified (있으면)
```

## README 필드 (필수만)

1. **anomaly** — 한 문단. 책상이 아니라 lived perception에서.
2. **내 첫 직관** — *계산 전에* 박는다. 에이전트 답 받기 전에. 빠지면 에이전트가 틀려도 인간이 모른다.
3. **surface 참조** — `data/surfaces.edn` (있으면)의 어떤 surface인가
4. **최소 데이터** — 이 study가 쓰는 출처 목록
5. **모델/계산** — Clojure 코드 또는 손계산 메모
6. **memo** — 가정 + 결과 + 불확실성
7. **직관 vs 모델 vs 실제** — 가능하면 셋 다. 빠진 칸도 빈 채로 적기.

## 수명

- study 폴더는 **append-only**. 정정은 하지 않고 새 라운드를 추가한다.
- 라운드 1, 라운드 2 식으로 누적. 처음에 틀린 직관도 그대로 둔다 — 그게 이 도구의 자산.
